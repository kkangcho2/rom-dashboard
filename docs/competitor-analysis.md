## 1. 경쟁사 개요 및 상세 분석

### 1.1 SocialBlade
- **URL**: https://socialblade.com
- **설립**: 2008년
- **포지션**: YouTube/Twitch/Instagram 통계 추적의 원조 플랫폼

#### 핵심 기능
- 채널별 구독자 수, 조회수 일별/월별/연간 추이 그래프
- 예상 수익 범위 (일/월/연 단위)
- 채널 등급 시스템 (A++ ~ F)
- 실시간 구독자 카운터 (Live Sub Count)
- 채널 간 비교 도구 (최대 3개 채널)
- Top Lists (카테고리별, 국가별 랭킹)
- Future Projections (구독자/조회수 예측)

#### 데이터 수집 방법
- **YouTube Data API v3** (공식 API): 구독자 수, 조회수, 동영상 수 등 기본 메트릭
- **자체 크롤링/트래킹**: 일별 스냅샷을 저장하여 히스토리 데이터 구축
- **수익 추정**: CPM 범위를 적용한 자체 알고리즘 (공식 데이터 아님)
- 데이터 업데이트 주기: 일 1회 (유료 플랜은 더 빈번)

#### 표시 메트릭
| 메트릭 | 데이터 소스 | 실제/추정 |
|--------|------------|-----------|
| 구독자 수 | YouTube API | 실제 (공개 설정 시) |
| 총 조회수 | YouTube API | 실제 |
| 동영상 수 | YouTube API | 실제 |
| 일별 구독자 변화 | 자체 트래킹 | 실제 (일별 스냅샷 차이) |
| 일별 조회수 변화 | 자체 트래킹 | 실제 (일별 스냅샷 차이) |
| 예상 수익 | 자체 알고리즘 | 추정 (CPM 기반) |
| 채널 등급 | 자체 알고리즘 | 추정 |
| 미래 예측 | 자체 알고리즘 | 추정 |

#### UI/UX 패턴
- **레이아웃**: 단일 채널 중심 페이지, 상단에 요약 카드 + 하단에 상세 테이블/그래프
- **채널 카드**: 프로필 이미지 + 채널명 + 등급 배지 + 핵심 수치 3-4개
- **차트**: 선형 그래프 (30일/60일/90일/365일 토글), 일별 막대그래프
- **색상**: 증가=초록, 감소=빨강 (델타 표시)
- **테이블**: 일별 통계 테이블 (날짜, 구독자, 조회수, 변화량)
- **Empty State**: "No data available for this period" 텍스트만 표시

#### 가격
- 무료: 기본 통계, 광고 포함
- Bronze ($3.99/월): 광고 제거, 빠른 업데이트
- Silver ($9.99/월): 고급 통계, 커스텀 대시보드
- Gold ($39.99/월): API 접근, 대량 데이터

- **자체 크롤링**: 일별 데이터 수집 및 히스토리 구축

- **광고주 데이터**: CPM/CPV 시장 데이터 기반 가치 산정

- Business ($49/월~): 고급 분석, 검색 필터 확장, 리포트 내보내기
- Enterprise (문의): 대량 API, 커스텀 솔루션

- **자체 AI/ML 엔진**: 팔로워 품질 분석, 봇 감지, 인구통계 추정
- **행동 패턴 분석**: 참여 패턴의 이상치 탐지로 가짜 팔로워 식별

### 2.1 YouTube Data API v3 (공통 기반)

모든 플랫폼이 사용하는 핵심 API. 제공되는 실제 데이터:

```
공개 데이터 (API Key만 필요):
- channel.statistics: subscriberCount, viewCount, videoCount
- video.statistics: viewCount, likeCount, commentCount
- video.snippet: title, description, tags, categoryId, publishedAt
- video.contentDetails: duration
- search: 키워드/채널/동영상 검색
- playlistItems: 채널의 업로드 목록

비공개 데이터 (OAuth 필요 - 채널 소유자 동의):
- analytics: 시청 시간, 인구통계, 트래픽 소스, 수익
- 이 데이터는 채널 소유자만 접근 가능
```

### 2.2 각 플랫폼의 추가 데이터 수집

| 방법 | 사용 플랫폼 | 수집 데이터 |
|------|-----------|------------|
| 일별 스냅샷 트래킹 | SocialBlade, Nox, Playboard | 구독자/조회수 일별 변화 |
| 라이브챗 크롤링 | Playboard | 슈퍼챗 금액 |
| 댓글 분석 | HypeAuditor | 봇/가짜 팔로워 감지 |
| ML/AI 추정 | HypeAuditor, Nox, CreatorIQ | 인구통계, 참여 품질 |
| OAuth 연동 | CreatorIQ, Upfluence | 1st party 데이터 |
| 컴퓨터 비전 | Stream Hatchet | 방송 화면 내 브랜드 인식 |

### 2.3 핵심 인사이트: "실제 데이터" vs "추정 데이터"

```
확실히 실제인 데이터:
  - 구독자 수 (YouTube API, 공개 설정 시)
  - 총 조회수 (YouTube API)
  - 동영상 수 (YouTube API)
  - 개별 동영상 조회수/좋아요/댓글 수 (YouTube API)
  - 동영상 메타데이터 (제목, 설명, 태그, 게시일)

추정인 데이터 (그러나 업계에서 표준적으로 표시):
  - 예상 수익/채널 가치 (CPM 기반 추정)
  - 시청자 인구통계 (ML 추정)
  - 참여율 (좋아요+댓글 / 조회수로 계산 - 실제 데이터 기반 계산)
  - 성장률 (일별 스냅샷 차이 - 실제 데이터 기반 계산)
  - 채널 등급/점수 (자체 알고리즘)
```

## 4. LivePulse 포지셔닝 분석

### 4.1 경쟁 구도에서의 차별점

LivePulse의 고유한 포지션: **"모바일 게임" 특화 YouTube 크리에이터 분석**

| 영역 | 기존 경쟁사 | LivePulse 기회 |
|------|-----------|---------------|
| 게임 특화 분류 | 대부분 "Gaming" 하나로 통합 | 모바일 게임 타이틀별 세분화 |
| 모바일 게임 메타데이터 | 없거나 빈약 | 앱스토어/플레이스토어 연동 |
| 게임별 크리에이터 매칭 | Gamesight만 부분 제공 | 모바일 게임 타이틀 <-> 크리에이터 매핑 |
| 한국/아시아 시장 | Playboard(한국), Nox(중국) | ROM 기반 + 한국 모바일 게임 특화 |
| 광고주 도구 | 대부분 범용적 | 모바일 게임 퍼블리셔를 위한 맞춤 도구 |

### 4.2 LivePulse가 반드시 갖춰야 할 "실제 데이터" 기반 기능

**1단계 - 최소 필수 (YouTube API로 가능)**
- [ ] 채널 기본 정보 (구독자, 조회수, 동영상 수)
- [ ] 동영상 목록 및 개별 성과 (조회수, 좋아요, 댓글)
- [ ] 동영상 메타데이터 기반 게임 태깅 (제목/설명에서 게임명 추출)
- [ ] 검색/필터 (게임명, 구독자 범위, 국가)

**2단계 - 차별화 (일별 트래킹 필요)**
- [ ] 구독자/조회수 일별 변화 추적
- [ ] 성장률 계산 및 트렌드 표시
- [ ] 게임별 크리에이터 랭킹

**3단계 - 고급 기능 (추가 데이터 소스)**
- [ ] 앱스토어/플레이스토어 게임 데이터 연동
- [ ] 참여율 분석 및 벤치마킹
- [ ] 예상 수익/채널 가치 (CPM 기반, "추정"임을 명시)
- [ ] 크리에이터 비교 도구

## 6. 핵심 시사점 및 권장 사항

### 6.1 데이터 전략
1. **YouTube Data API v3를 핵심 데이터 소스로** - 모든 경쟁사의 기반
2. **일별 크롤링/스냅샷 필수** - 성장률 데이터는 경쟁사 대비 필수 기능
3. **"추정" 데이터는 명확히 라벨링** - 수익 추정 등은 "Estimated" 배지 표시
4. **게임 태깅 자동화** - 동영상 제목/설명에서 게임명을 NLP로 추출하는 것이 핵심 차별점
5. **앱스토어 데이터 연동** - 게임 메타데이터(장르, 퍼블리셔, 다운로드수)를 크리에이터 데이터와 결합

### 6.2 UI/UX 전략
1. **카드 기반 레이아웃 채택** (업계 표준)
2. **필터 시스템 강화** - 게임 타이틀, 국가, 구독자 범위, 참여율 범위
3. **Empty State를 기회로** - 데이터 수집 중 상태를 투명하게 표시, 사용자가 크리에이터 추가 요청 가능하도록
4. **모바일 최적화** - 경쟁사 대부분 데스크톱 중심, 모바일 UX로 차별화 가능
5. **다크 모드 지원** - Playboard의 다크 모드가 게이밍 커뮤니티에서 인기

### 6.3 우선순위 로드맵
```
Phase 1 (MVP): YouTube API 기반 크리에이터 검색 + 기본 프로필 + 게임 태깅
Phase 2: 일별 트래킹 + 성장률 + 랭킹 시스템
Phase 3: 참여율 분석 + 크리에이터 비교 + 광고주 도구
Phase 4: AI 기반 추정 (인구통계, 수익) + 캠페인 관리
```
